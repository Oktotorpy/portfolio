# Portfolio CMS — Server Setup Guide

Complete instructions for deploying the Portfolio CMS on a fresh Ubuntu 24.04 VPS.

## Prerequisites

- Ubuntu 24.04 VPS with root SSH access
- A domain name (e.g. `vitz.pro`)
- A GitHub repository with the portfolio code

## Architecture

```
Internet → Caddy (HTTPS, port 443)
              ├── /api/*     → Gunicorn/Flask (port 8000)
              ├── /uploads/* → Static files from /opt/portfolio/uploads
              └── /*         → SvelteKit/Node (port 3000)
```

## Step 1: Initial Server Setup

SSH into your server as root and download the deploy script:

```bash
ssh root@YOUR_SERVER_IP
```

Create the app directory and place the deploy script:

```bash
mkdir -p /opt/portfolio
# Copy deploy.sh to /opt/portfolio/ (or curl it from your repo)
chmod +x /opt/portfolio/deploy.sh
```

Run the setup command:

```bash
/opt/portfolio/deploy.sh setup
```

This installs all dependencies:
- **Python 3** + venv + pip
- **Node.js** + npm
- **SQLite3**
- **Caddy** (web server with automatic HTTPS)
- **Git**, **gettext** (for envsubst)

It also creates:
- System user `portfolio` (no login shell)
- Directory structure at `/opt/portfolio/`
- Python virtual environment with Flask + Gunicorn
- Template `.env` file with a generated `SECRET_KEY`

## Step 2: Connect to GitHub

```bash
/opt/portfolio/deploy.sh git-init git@github.com:YOUR_USER/YOUR_REPO.git
```

On the first run, this generates an SSH deploy key and prints the public key. Add it to your GitHub repo:

1. Go to your repo → **Settings** → **Deploy keys** → **Add deploy key**
2. Paste the key, give it a name, click **Add key**
3. Run the command again:

```bash
/opt/portfolio/deploy.sh git-init git@github.com:YOUR_USER/YOUR_REPO.git
```

This clones the repository to `/opt/portfolio/repo/`.

## Step 3: Configure Environment

Edit the `.env` file:

```bash
nano /opt/portfolio/.env
```

Set these values:

```
DOMAIN=vitz.pro
ADMIN_USERNAME=admin
ADMIN_PASSWORD_HASH=<generated hash>
SECRET_KEY=<auto-generated, leave as-is>
DB_PATH=/opt/portfolio/data/portfolio.db
UPLOAD_DIR=/opt/portfolio/uploads
CORS_ORIGIN=https://vitz.pro
BACKEND_PORT=8000
FRONTEND_PORT=3000
INTERNAL_API_URL=http://localhost:8000
```

### Generate the admin password hash

```bash
cd /opt/portfolio/repo
python3 deploy/generate_hash.py
```

Enter your desired password. Copy the hash output into `ADMIN_PASSWORD_HASH` in `.env`.

## Step 4: Configure DNS

At your domain registrar, add DNS records pointing to your server:

| Type | Name  | Value             | TTL |
|------|-------|-------------------|-----|
| A    | `@`   | `YOUR_SERVER_IP`  | 300 |
| A    | `www` | `YOUR_SERVER_IP`  | 300 |

Find your server IP:

```bash
curl -4 ifconfig.me
```

Verify DNS propagation (may take up to 30 minutes):

```bash
dig +short vitz.pro
```

## Step 5: Configure Firewall

```bash
ufw allow 22/tcp     # SSH
ufw allow 80/tcp     # HTTP (ACME challenges + redirect)
ufw allow 443/tcp    # HTTPS
ufw enable
ufw status
```

## Step 6: Deploy

```bash
/opt/portfolio/deploy.sh pull
```

This single command:
1. Pulls latest code from GitHub
2. Syncs backend files, installs Python dependencies
3. Syncs frontend files, runs `npm install` + `npm run build`
4. Generates Caddy config from template (with your domain)
5. Installs systemd service files
6. Initializes the database (first run only)
7. Restarts all services
8. Caddy automatically obtains a Let's Encrypt TLS certificate

After it completes, visit `https://vitz.pro` — the site should be live with HTTPS.

## Day-to-Day Operations

### Deploy updates

After pushing changes to GitHub:

```bash
ssh root@YOUR_SERVER_IP "/opt/portfolio/deploy.sh pull"
```

### Check status

```bash
/opt/portfolio/deploy.sh status
```

### View logs

```bash
/opt/portfolio/deploy.sh logs
```

### Restart services

```bash
/opt/portfolio/deploy.sh restart
```

### Stop services

```bash
/opt/portfolio/deploy.sh stop
```

## Server Directory Structure

```
/opt/portfolio/
├── deploy.sh          # Deployment script
├── .env               # Environment config (not in git)
├── repo/              # Git clone (read-only source)
├── backend/           # Deployed Flask app (rsynced from repo)
├── frontend/          # Deployed SvelteKit (rsynced + built)
├── uploads/           # User-uploaded media (not in git)
├── data/              # portfolio.db (not in git)
└── venv/              # Python virtualenv
```

## Troubleshooting

### Site unreachable

```bash
# Check all services are running
/opt/portfolio/deploy.sh status

# Check Caddy specifically
systemctl status caddy
journalctl -u caddy --since "10 minutes ago"
```

### HTTPS certificate not issued

- Ensure ports 80 and 443 are open: `ufw status`
- Ensure DNS points to correct IP: `dig +short vitz.pro`
- Check Caddy logs: `journalctl -u caddy -f`

### "Not authenticated" on admin login

The backend uses `--preload` with Gunicorn to share session state. If auth fails:

```bash
systemctl restart portfolio-backend
journalctl -u portfolio-backend --since "5 minutes ago"
```

### 502 Bad Gateway

Backend or frontend crashed:

```bash
journalctl -u portfolio-backend -f
journalctl -u portfolio-frontend -f
```

### CORS errors in browser

Ensure `CORS_ORIGIN` in `.env` matches your domain exactly including `https://`:

```
CORS_ORIGIN=https://vitz.pro
```

Then restart: `/opt/portfolio/deploy.sh restart`

### Database issues

The database auto-migrates on every backend start. To manually initialize:

```bash
cd /opt/portfolio/backend
sudo -u portfolio \
    DB_PATH=/opt/portfolio/data/portfolio.db \
    /opt/portfolio/venv/bin/python -c "from database import init_db; init_db()"
```

### Backups

Set up automated backups with the included script:

```bash
# Test it manually first
bash /opt/portfolio/repo/deploy/backup.sh

# Add to cron (daily at 3am)
crontab -e
# Add: 0 3 * * * /opt/portfolio/repo/deploy/backup.sh
```

Backups are saved to `/opt/portfolio/backups/` with 30-day retention.
